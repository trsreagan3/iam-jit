# Threat-feed test fixtures

For L6 — exercises Ed25519 signature verification + apply + revoke
cycle without ever touching the operator's real publisher trust
store.

## Test keypair

Stage B materializes a brand-new Ed25519 keypair for the test:

* `PUBLISHER-TEST-KEY-DO-NOT-USE-IN-PROD.priv` — committed
  intentionally; this is a TEST key with no production trust. The
  filename has the loud DO-NOT-USE warning so a future agent can't
  mistake it for an operator artifact.
* `L6-publisher.pub` — public half; pinned into the bouncer trust
  store at scenario start.

## Test payload

`L6-block-evil-host.signed.json` — a signed threat-feed bundle
that, when applied, adds a single deny rule for `evil.test.example`.
The bundle is regenerated by Stage B (`make fixtures` or similar)
whenever the test keypair changes.

## Tamper variant

`L6-block-evil-host.tampered.json` — same payload, body mutated by
one byte AFTER signing. Used to verify the apply path rejects
tampered bundles.
